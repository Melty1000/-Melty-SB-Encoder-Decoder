Help Guide

Documentation & Reference

Comprehensive technical reference for the Streamer.bot export format. Use this when generating or editing JSON manually.

1. Root Structure

{
    "meta": {
        "name": "Export Name",
        "author": "Author Name",
        "version": "1.0.0",
        "description": "Description"
    },
    "data": {
        "actions": [ ... ],
        "commands": [],
        "timers": [],
        "queues": [],
        "websocketServers": [],
        "websocketClients": []
    },
    "version": 23,
    "exportedFrom": "1.0.1",
    "minimumVersion": "1.0.0-alpha.1"
}

2. Encoder Requirements

To successfully encode C# scripts, your JSON template must be prepared correctly:

  • ByteCode Field: For C# actions (Type 99999), the byteCode field MUST contain the exact filename of the script (e.g., "MyScript.cs"), NOT the raw code.
  • Matching Filenames: The filename in byteCode must exactly match the name of the uploaded .cs file.
  • Sub-Action Names: Ensure every sub-action has a name field. This is used by the Decoder to name the extracted files.
  • LOWERCASE GUIDs: All UUIDs (id, parentId, queue, etc.) MUST be lowercase. Uppercase GUIDs will cause import failure.
  • Required Arrays: The data object must contain commands, timers, queues, etc., even if empty.

Important: If your JSON contains raw code in byteCode, the encoder will not be able to inject the scripts. Use the filename instead.

Validation Checklist
  • ✅ UUIDs: Must be RFC4122 format (lowercase with hyphens).
  • ✅ Booleans: Never use null (always true/false).
  • ✅ Arrays: Never use null (use [] for empty).
  • ✅ Index: Sequential 0, 1, 2... within same parent.
  • ✅ ParentId: Must reference existing UUID or be null.

3. Action Structure

Each action in data.actions[] has these fields:

  • id: Unique GUID identifier (MUST be lowercase)
  • name: Display name in Streamer.bot
  • group: Category/folder name
  • enabled: Boolean to enable/disable action
  • queue: Queue ID for execution order (use 00000000-0000-0000-0000-000000000000 for default)
  • alwaysRun: If true, runs even if action is disabled
  • concurrent: If true, allows multiple instances to run simultaneously
  • randomAction: If true, picks random sub-action to execute
  • triggers[]: Array of events that start this action
  • subActions[]: Array of steps to execute
  • collapsedGroups[]: Array of group IDs (required, can be empty)

6. Nested Sub-Actions

Logic groups (If/Else, Random) contain nested subActions[]. Use parentId to link children to their parent group.

{
  "type": 120,
  "input": "%firstMessage%",
  "operation": 0,
  "value": "false",
  "subActions": [
    {
      "type": 99901,
      "parentId": "parent-guid",
      "subActions": [...]
    },
    {
      "type": 99902,
      "parentId": "parent-guid",
      "subActions": []
    }
  ]
}

7. Variables

Use %variableName% syntax in text fields. Common variables:

  • %userName%, %user% - Triggering user's name
  • %msgId% - Message ID for replies
  • %targetUserProfileImageUrl% - User's profile image
  • %firstMessage% - Boolean for first-time chatters
  • %targetUser% - Target user's name
  • %game% - Current game name
  • %inputUserName% - User input from argument

      "subActions": [
        {
          "id": "sub-1-lowercase",
          "type": 120,
          "input": "%firstMessage%",
          "operation": 0,
          "value": "false",

          "parentId": null,
          "subActions": [
            {
              "id": "if-branch-lowercase",

                "id": "msg-lowercase",

            }

        }
      ],

    "commands": [],
    "timers": [],
